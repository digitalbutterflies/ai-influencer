====================================================
  AI INFLUENCER STUDIO - SETUP AND DEPLOYMENT GUIDE
====================================================


PRODUCTION
--------------------------------------------------
Primary URL:
   https://influencers.2zero.network

Worker URL:
   https://ai-influencer.jens-e4b.workers.dev

Health check:
   https://influencers.2zero.network/api/health


WHAT YOU NEED
--------------------------------------------------
- Node.js 18 or newer
- PowerShell
- Cloudflare access through Wrangler
- A Higgsfield account for image and video generation


LOCAL DEVELOPMENT
--------------------------------------------------
1. Install dependencies:

      npm install

2. Start the Vite app:

      npm run dev

3. Open:

      http://localhost:5173

Do not stop an existing Vite dev server on port 5173 unless you intentionally started it.


CONNECTING HIGGSFIELD
--------------------------------------------------
1. Open Settings.
2. Click "Connect Higgsfield".
3. Log in with your Higgsfield account.
4. The app uses your own Higgsfield credits.

Higgsfield tokens are stored in browser localStorage per domain.
After switching from the workers.dev domain to influencers.2zero.network,
connect Higgsfield again on influencers.2zero.network.


CLOUDFLARE DEPLOYMENT
--------------------------------------------------
1. Check Cloudflare authentication:

      npm run cf:whoami

2. Build locally:

      npm run build

3. Validate deployment without publishing:

      npm run deploy:dry-run

4. Deploy the Worker and static assets:

      npm run deploy

The Wrangler config runs npm run build, uploads dist as Workers Static Assets,
serves SPA routes through index.html fallback, and sends /api/* through
worker/index.js.


POST-DEPLOY CHECKS
--------------------------------------------------
Check the production health endpoint:

      Invoke-WebRequest -Uri "https://influencers.2zero.network/api/health" -UseBasicParsing

Check the primary SPA route:

      Invoke-WebRequest -Uri "https://influencers.2zero.network/settings" -UseBasicParsing

Expected health body:

      {"ok":true,"runtime":"cloudflare-workers","media":{"images":"not_configured","stream":"not_configured"}}


LEGACY LOCALSTORAGE IMPORT
--------------------------------------------------
If user-created avatars only exist on the old workers.dev origin, open:

      https://influencers.2zero.network/?importLegacy=1

This imports app data from:

      https://ai-influencer.jens-e4b.workers.dev

It does not import Higgsfield tokens, Anthropic API keys, OAuth verifier data,
or refresh tokens. Reconnect Higgsfield on the production domain after import.


MEDIA STATUS
--------------------------------------------------
- Cloudflare Images is not active yet.
- Cloudflare Stream is not active yet.
- Current generated media is local browser data, static seed assets, or Higgsfield-hosted URLs.
- Do not describe current generated media as persisted in Cloudflare Images or Stream.

NOTES
--------------------------------------------------
- All user data is browser-local unless it is a static seed asset.
- Clearing browser data resets the local app data.
- Image and video generation needs internet access for Higgsfield.
- No secrets belong in wrangler.jsonc or source files.
